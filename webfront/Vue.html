                # vue的生命周期

                当一个new Vue()的时候，就表示vue的生命周期开始了，然后伴随着程序不断的进行，最终走向消亡。

                生命周期的vue实例化阶段

                1. beforeCreate：第一个钩子函数：此阶段实例刚在内存中创建出来，data和method都没有准备好。

                  用途：给页面渲染过程中加一些动画（loading事件）

                2. created：*第二个钩子函数：此阶段data和method都准备完毕了，只是还没有开始编译模板。

                  用途：通过Ajax请求数据和处理

                3. beforeMount：第三个钩子函数：此阶段完成了虚拟DOM模板的编译，但是还没有挂在到页面中。

                  用途：

                4. mounted：*第四个钩子函数：此阶段标志着vue实例化阶段完全结束了，此时模板也已经渲染结束,元素会挂在到html中。

                  用途：关闭渲染动画

                生命周期的运行阶段

                1. beforeUpdate：如果data发生改变，触发此钩子函数，data改变了，但是页面的DOM元素还没有改变,只是在虚拟DOM中发生了改变。
                2. updated：如果data发生改变，触发此钩子函数，data改变了，页面的DOM元素也重新渲染了，虚拟DOM对页面进行了覆盖。

                生命周期的销毁阶段

                1. beforeDestroy：实例即将销毁，该阶段实例还是可以使用。
                2. destroyed：实例已经销毁，data和methods都无法使用

                # 计算属性（computed）

                当我们对变量需要进行计算的时候需要将计算写在计算属性中：

                简单使用

                <div id="app">
    <!-- 计算属性只调用了一次，因为有缓存 -->
    <h3>平均成绩：{{getAvgProperty}}</h3>
    <h3>平均成绩：{{getAvgProperty}}</h3>
</div>
<script>
    const vue = new Vue({
        el: '#app',
        data: {
            sorce: [88, 78, 98, 60]
        },
        //计算属性：涉及到复杂的业务计算时候，必须使用计算属性
        computed: {
            getAvgProperty(){
                console.log('计算平均值调用了');
                return this.sorce.reduce((prev, item) => prev + item)/this.sorce.length;
            }
        }
    })
</script>

                计算属性VS函数的区别：计算属性会对计算结果进行缓存，当多次访问计算属性的时候，不必再次执行计算函数。

                set和get都使用（当给计算数学赋值的时候会走set，例如：AvgProperty='123'）

                <div id="app">
    <!-- 计算计算属性 -->
    <h3>平均成绩：{{AvgProperty}}</h3>
    {{value}}
</div>
<script>
    const vue = new Vue({
        el: '#app',
        data: {
            sorce: [88, 78, 98, 60],
            value:''
        },
        //计算属性：涉及到复杂的业务计算时候，必须使用计算属性
        computed: {
            AvgProperty:{
                get(){
                    console.log('get')
                    return this.sorce.reduce((prev, item) => prev + item) / 4;
                },
                set(value){
                    console.log('set');
                    this.value=value;
                }
            }
        }
    })
</script>

                # 侦听器（watch）

                监控状态数值改变

                虽然计算属性在大多数情况下更合适，但有时也需要一个自定义的侦听器。这就是为什么 Vue 通过 watch 选项提供了一个更通用的方法，来响应数据的变化。当需要在数据变化时执行异步或开销较大的操作时，这个方式是最有用的。

                使用

                <div id="app">
    {{number}}
    {{student}}
</div>
<script>
    const vue=new Vue({
        el:'#app',
        //状态
        data:{
            number:100,
            student:{
                name:'张三',
                age:18
            }
        },
        //侦听器watch
        watch:{
            //可以不加引号
            number:function(){
                console.log('number改变了');
            },
            //加引号是为对象中属性准备的
            'student.name':function(){
                console.log('name改变了');
                setTimeout(() => {
                    console.log(this.student.name);
                }, 1000);
            }
        }
    })
</script>

                计算属性和侦听器的区别：

                • *计算属性不支持里面执行异步代码，侦听器中可以执行异步代码
                • 计算属性支持缓存，侦听器不支持缓存

                tips：不涉及异步的大部分情况都是用计算属性更合适，用到异步的需要使用侦听器

                # 过滤器（filters）

    })
</script>

                # 组件中的data必须是个函数

                在javascript中，当函数运行时，会在内存中有一个新的空间，空间内的属性值是独立的。

                以对象方式创建时，每个对象获取到的是一个内存的地址0x23，所以当地址中的值改变，所有对象都会跟着发生改变。

                原因是在javascript只有函数才有自己的作用域，组件要避免其他外部影响我们内部的data

                注意：

                1.组件内部是不能访问vue实例里边的数据的，组件是一个单独独立功能模块的封装，有自己的html模板 也应该有自己的data。

                2.组件的data必须是一个函数为了区分开组件之间的区别,需要使用函数来返回不同的对象。

                # 父子组件的通讯

                组件中的数据，不应该写在组件内部，写在内部就是写死了，消失复用性。组件中的数据要有调用者传进来

                在开发中，往往一些数据需要从上层传递到下层：

                比如在一个页面中，我们从服务器请求到了很多数据。

                其中一部分数据，并非是我们整个页面的大组件来展示的，而是需要下面的子组件进行展示。

                这个时候，并不会让子组件发次发送一个网络请求，而是直接让大组件（父组件）将数据传递给小组件（子组件）。

                父传子（props）

                使用

                <div id="app">
    <!--通过属性的方式给组件传值-->
    <menucpn :clist="list"></menucpn>

                # loader

                webpack 只能理解 JavaScript 和 JSON 文件，这是 webpack 开箱可用的自带能力。loader 让 webpack 能够去处理其他类型的文件，并将它们转换为有效 模块 (opens new window)，以供应用程序使用，以及被添加到依赖图中。

                简而言之，由于webpack天生只认识javascript和json，那么如果我们想把其他资源文件（css、less、sass、img...）打包到webpack中，那么我们就需要对loader进行配置。

                loader的两个属性

                1. test 属性，识别出哪些文件会被转换。
                2. use 属性，定义出在进行转换时，应该使用哪个 loader。
                //webpack.config.js
const path = require('path');

module.exports = {
    output: {
        filename: 'my-first-webpack.bundle.js',
    },
    // loader配置
    module: {
        rules: [{ 
            test: /\.txt$/,
            use: 'raw-loader'
        }],
    },
};

                “嘿，webpack 编译器，当你碰到「在 require()/import 语句中被解析为 '.txt' 的路径」时，在你对它打包之前，先 use(使用) raw-loader 转换一下。”

                # 热加载配置

                模块热替换(HMR - hot module replacement)功能会在应用程序运行过程中，替换、添加或删除 模块 (opens new window)，而无需重新加载整个页面。主要是通过以下几种方式，来显著加快开发速度：

                • 保留在完全重新加载页面期间丢失的应用程序状态。
                • 只更新变更内容，以节省宝贵的开发时间。
                • 在源代码中 CSS/JS 产生修改时，会立刻在浏览器中进行更新，这几乎相当于在浏览器 devtools 直接更改样式。

                配置步骤

                # keep-alive

                组件保持，组件缓存，当跳转到其他页面再回到该页面，该页面不会重新加载，之前的数据（文本框）还会存在

                使用：用keep-alive将要缓存的组件包裹起来，不写参数是所有组件都缓存

                <keep-alive>
    <router-view></router-view>
</keep-alive>

                参数设置

                include是设置需要缓存的组件；

                exclude是设置除了某些组件都缓存；

                max是设置最多可以缓存多少组件实例一旦这个数字达到了，在新实例被创建之前，已缓存组件中最久没有被访问的实例会被销毁掉。

                nclude 和 exclude二者都可以用逗号分隔字符串、正则表达式或一个数组来表示：

                <!-- 逗号分隔字符串 	是组件的名称(组件中name值)-->
<keep-alive include="a,b">
    <component :is="view"></component>
</keep-alive>

<!-- 正则表达式 (使用 `v-bind`) -->
<keep-alive :include="/a|b/">
    <router-view></router-view>
</keep-alive>

<!-- 数组 (使用 `v-bind`) -->
<keep-alive :include="['a', 'b']">
    <component :is="view"></component>
</keep-alive>

<keep-alive :max="10">
	<router-view></router-view>
</keep-alive>

                # 导航守卫

                vue-router 提供的导航守卫主要用来通过跳转或取消的方式守卫导航。这里有很多方式植入路由导航中：全局的，单个路由独享的，或者组件级的。

                # 全局守卫

                全局前置守卫

                在路由配置文件中使用router.beforeEach 注册一个全局前置守卫：

                const router = new VueRouter({
    //配置路由和组件之间的对应关系
    routes: [
        {
            path: '/bar',
            component: bar,
        }
    ],
    mode: 'history'
})


//				目标网页,从哪个网页来,进行下一步操作
router.beforeEach((to, from,next) => {
    console.log('前置守卫');
    //让其通过
    next();
    // ...
    // 返回 false 以取消导航
    return false
})

                全局后置守卫

                你也可以在路由配置文件中注册全局后置钩子，然而和守卫不同的是，这些钩子不会接受 next 函数也不会改变导航本身：

                router.afterEach((to, from) => {
    console.log('全局后置守卫');
})

                # 路由独享守卫

                在路由配置文件中的路由规则中配置

                const router = new VueRouter({
    //配置路由和组件之间的对应关系
    routes: [
        {
            path: '/bar',
            component: bar,
            beforeEnter:(to,from,next)=>{
                console.log('路由独享守卫');
                next();
                return false;
            }
        }
    ],
    mode: 'history'
})

                tips：beforeEnter 守卫 只在进入路由时触发，不会在 params、query 或 hash 改变时触发。例如：从/users/1进入到/users/2

                # 组件内守卫

                你可以为路由组件添加以下配置：

                • beforeRouteEnter
                • beforeRouteUpdate
                • beforeRouteLeave
                export default {
    name:'',
    props:[''],
    data(){
      return {}  
    },
	beforeRouteEnter (to, from, next) {
        // 在渲染该组件的对应路由被验证前调用
        // 不能获取组件实例 `this` ！
        // 因为当守卫执行时，组件实例还没被创建！
		console.log('beforeRouteEnter');
		next();
	},
	beforeRouteUpdate(to, from,next) {
        // 在当前路由改变，但是该组件被复用时调用
        // 举例来说，对于一个带有动态参数的路径 `/users/:id`，在 `/users/1` 和 `/users/2` 之间跳转的时候，
        // 由于会渲染同样的 `UserDetails` 组件，因此组件实例会被复用。而这个钩子就会在这个情况下被调用。
        // 因为在这种情况发生的时候，组件已经挂载好了，导航守卫可以访问组件实例 `this`
        console.log('beforeRouteUpdate');
		next();
  },
  beforeRouteLeave(to, from,next) {
      // 在导航离开渲染该组件的对应路由时调用
      // 与 `beforeRouteUpdate` 一样，它可以访问组件实例 `this`
      console.log('beforeRouteLeave');
      next();
  },
}

                完整的导航解析流程

                1. 全局前置守卫beforeEach
                2. 路由独享守卫beforeEnter
                3. 组件守卫beforeRouteEnter
                4. 全局后置守卫afterEach
                5. 在重用的组件被调用的时候beforeRouteUpdate
                6. 离开该页面时调用beforeRouteLeave

                # getters

                只要是对state中的值进行处理的，都需要写在getters中。

                getters的作用主要是让我们的页面对数据进行筛选，将筛选后的结果进行返回显示。

                虽然每个vue中具有computed属性可以进行数据的计算，但是当使用较多的情况下，维护性就会比较差。

                使用

                配置文件中写筛选的方法

                const store = new Vuex.Store({
    //存储值
    state: {
        userlist: [{name:'张三',age:19},{name:'李四',age:17},{name:'王五',age:20}]
    },
    //处理值的方法
    getters: {
        //默认两个参数，一个state获取存储的值，一个getters获取处理的方法
		userfilter(state,getters){
            return state.userlist.filter(x=>x.age>18);
        }
    }
})

                vue组件中调用

                $store.getters.userfilter

                tips：getters中的对象是以属性方式调用，所以外部没有办法传递参数进来。如果需要传递参数怎么办？

                由于getters中只有两个参数，所以在外部调用需要向getters中传递值时，需要对getters中的函数进行闭包处理

                //配置文件中的方法
userfilter(state,getters){
    return function(age){
        return state.userlist.filter(x=>x.age>age);
	}
}

//vue中调用
$store.getters.userfilter(19)

                # actions

                mutation中调用的是同步函数，Action类似于Mutation,但是他是用来代替Mutation进行异步操作的

                通常情况下,vuex要求我们mutation中的方法必须是同步方法。主要原因是当我们使用devtools时，可以帮助我们捕捉到mutation的快照

                但是如果是异步操作，那么devtools将不能很好的追踪到这个操作。

                使用

                配置项中写异步方法：只是执行异步的操作，里面的同步修改state值需要调用mutations中的方法

                const store = new Vuex.Store({
    //存储值
    state: {
        count:10
    },
    //更改值的方法
    mutations: {
        insertcount(state){
            state.count++;
        }
    },
    //异步执行的方法
    actions: {
		asyncUpdate(context){
            setTimeout(() => {
                context.commit('insertcount')
            }, 1000);
        }
    },
})

                vue组件中调用

                //无参调用
$store.dispatch('asyncUpdate')

//有参调用
this.$store.dispatch('asyncUpdate','参数')

                tips：如果需要异步执行完有异步回调函数，需要使用promise进行封装

                //actions中的函数
asyncUpdate(context){
    return new Promise(resolve=>{
        setTimeout(() => {
            context.commit('insertcount');
            resolve('完毕');
        }, 1000);
    })
}

//vue组件中调用
this.$store.dispatch('asyncUpdate').then(res=>{console.log(res);})
